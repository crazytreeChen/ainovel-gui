## 二、P0: 标题一致性检查

### 问题

Writer 生成的章节标题不遵循大纲规划的标题，Editor 审阅时也不检查标题是否与大纲一致。

### 根因

| 环节 | 现状 | 问题 |
|------|------|------|
| 大纲 | `OutlineEntry.Title` 已规划 | ✅ 有规划 |
| Writer 上下文 | `current_chapter_outline` 传入 | ✅ 有传递 |
| writer.md | 说"根据本章核心主题提炼" | ❌ 没有要求沿用大纲标题 |
| plan_chapter 工具 | 不校验标题 | ❌ 无校验 |
| editor.md | `title_score` 只检查质量 | ❌ 不检查一致性 |

### 方案：建议一致，允许偏离

#### 2.1 修改 writer.md（第 49-54 行）

```markdown
## 章节标题

**优先沿用大纲标题**：`current_chapter_outline.title` 是大纲规划的标题，创作时优先使用。
如果大纲标题不适合本章实际内容（如大纲规划后情节调整），可以偏离，但必须在
`plan_chapter` 的 `notes` 中说明偏离理由。

正确：`title: "雨夜追凶"`（沿用大纲标题）
正确：`title: "暗夜追踪"`（偏离，notes: "原大纲标题'雨夜追凶'与本章实际场景不符，改为暗夜追踪"）
错误：`title: "主角在长老的密室里发现上古遗迹的秘密并决定独自前往"`（摘要式长句）
```

#### 2.2 修改 editor.md（一致性维度）

在 `维度一：设定一致性（consistency）` 中增加：

```markdown
#### 维度一：设定一致性（consistency）
- 事件顺序是否与时间线矛盾
- 世界规则边界是否被违反
- 角色属性是否前后矛盾
- 角色状态描述是否与 state_changes 记录一致
- 注意角色别名，同一人不同称呼不要误判
- **标题一致性**：检查 `chapter_plan.title` 是否与 `current_chapter_outline.title` 一致。
  - 如果一致，无需特别说明
  - 如果不一致，检查 `chapter_plan.notes` 是否有偏离理由
  - 如果不一致且无理由，在 issues 中标记为 warning，并在 consistency 维度 comment 中说明
```

#### 2.3 修改 plan_chapter.go（可选，增强校验）

在 `Execute` 中增加标题偏离警告（不阻断，仅记录）：

```go

if currentEntry, err := t.store.Outline.GetChapterOutline(plan.Chapter); err == nil && currentEntry != nil {
    if plan.Title != currentEntry.Title && plan.Notes == "" {
        // 标题偏离且无理由，记录警告（不阻断）
        log.Warn("标题偏离大纲", "chapter", plan.Chapter, "outline", currentEntry.Title, "plan", plan.Title)
    }
}
```

## 三、P0: 按严重程度排序 pending_rewrites

### 问题

当前 `PendingRewrites []int` 只存储章节号，按章节号排序。最严重的问题（critical）可能排在最后处理。

### 方案

将 `PendingRewrites` 从 `[]int` 改为 `[]PendingRewrite`，包含章节号和严重程度。

#### 2.1 修改 domain/runtime.go

```go
// PendingRewrite 待重写/打磨的章节信息
type PendingRewrite struct {
    Chapter  int    `json:"chapter"`
    Severity string `json:"severity"` // critical / error / warning
    Reason   string `json:"reason,omitempty"`
}

// Progress 中的字段变更
PendingRewrites   []PendingRewrite `json:"pending_rewrites,omitempty"`
```

#### 2.2 修改 store/progress.go

```go
// SetPendingRewrites 设置待重写章节队列（按严重程度排序）
func (s *ProgressStore) SetPendingRewrites(chapters []PendingRewrite, reason string) error {
    return s.io.WithWriteLock(func() error {
        p, err := s.loadUnlocked()
        if err != nil {
            return err
        }
        if p == nil {
            return nil
        }
        normalized, err := normalizePendingRewrites(chapters, p.CompletedChapters)
        if err != nil {
            return err
        }
        // 按严重程度排序：critical > error > warning
        sortPendingRewrites(normalized)
        p.PendingRewrites = normalized
        p.RewriteReason = reason
        return s.saveUnlocked(p)
    })
}

// sortPendingRewrites 按严重程度排序
func sortPendingRewrites(rewrites []PendingRewrite) {
    severityOrder := map[string]int{
        "critical": 0,
        "error":    1,
        "warning":  2,
    }
    slices.SortFunc(rewrites, func(a, b PendingRewrite) int {
        sa, sb := severityOrder[a.Severity], severityOrder[b.Severity]
        if sa != sb {
            return sa - sb
        }
        return a.Chapter - b.Chapter // 同级按章节号
    })
}

// CompleteRewrite 从待重写队列中移除已完成的章节
func (s *ProgressStore) CompleteRewrite(chapter int) error {
    return s.io.WithWriteLock(func() error {
        p, err := s.loadUnlocked()
        if err != nil {
            return err
        }
        if p == nil {
            return nil
        }
        var remaining []PendingRewrite
        for _, pr := range p.PendingRewrites {
            if pr.Chapter != chapter {
                remaining = append(remaining, pr)
            }
        }
        p.PendingRewrites = remaining
        if len(remaining) == 0 {
            if err := domain.ValidateFlowTransition(p.Flow, domain.FlowWriting); err != nil {
                return err
            }
            p.Flow = domain.FlowWriting
            p.RewriteReason = ""
        }
        return s.saveUnlocked(p)
    })
}
```

## 三、P0: 重写后自动触发审阅

### 问题

重写完成后需等下一轮循环才被审阅，质量闭环慢。

### 方案

在 `CompleteRewrite` 中检测队列清空后，自动标记需要立即审阅。

#### 3.1 修改 domain/runtime.go

```go
// Progress 新增字段
ImmediateReviewChapters []int `json:"immediate_review_chapters,omitempty"` // 需要立即审阅的章节
```

#### 3.2 修改 store/progress.go

```go
// CompleteRewrite 从待重写队列中移除已完成的章节
func (s *ProgressStore) CompleteRewrite(chapter int) error {
    return s.io.WithWriteLock(func() error {
        p, err := s.loadUnlocked()
        if err != nil {
            return err
        }
        if p == nil {
            return nil
        }
        var remaining []PendingRewrite
        for _, pr := range p.PendingRewrites {
            if pr.Chapter != chapter {
                remaining = append(remaining, pr)
            }
        }
        p.PendingRewrites = remaining
        
        // 标记需要立即审阅
        p.ImmediateReviewChapters = append(p.ImmediateReviewChapters, chapter)
        
        if len(remaining) == 0 {
            if err := domain.ValidateFlowTransition(p.Flow, domain.FlowWriting); err != nil {
                return err
            }
            p.Flow = domain.FlowWriting
            p.RewriteReason = ""
        }
        return s.saveUnlocked(p)
    })
}

// PopImmediateReviewChapter 弹出一个需要立即审阅的章节
func (s *ProgressStore) PopImmediateReviewChapter() (int, error) {
    var chapter int
    err := s.io.WithWriteLock(func() error {
        p, err := s.loadUnlocked()
        if err != nil {
            return err
        }
        if p == nil || len(p.ImmediateReviewChapters) == 0 {
            return nil
        }
        chapter = p.ImmediateReviewChapters[0]
        p.ImmediateReviewChapters = p.ImmediateReviewChapters[1:]
        return s.saveUnlocked(p)
    })
    return chapter, err
}
```

#### 3.3 修改 host/flow/router.go

```go

    return &Instruction{
        Agent:  "editor",
        Task:   fmt.Sprintf("审阅第 %d 章（重写后自动触发）", ch),
        Reason: "重写完成后自动触发审阅",
    }
}
```

## 四、P1: 智能修复策略选择

### 问题

polish 固定用 edit_chapter，rewrite 固定用 draft_chapter，不够精准。

### 方案

在 save_review 中根据问题类型自动选择修复策略。

#### 4.1 修改 domain/review.go

```go
// ReviewEntry 新增字段
RepairStrategy string `json:"repair_strategy,omitempty"` // auto / edit / rewrite
```

#### 4.2 修改 tools/save_review.go

```go
// selectRepairStrategy 根据问题类型选择修复策略
func selectRepairStrategy(review *domain.ReviewEntry) string {
    criticalCount := 0
    errorCount := 0
    warningCount := 0
    
    for _, issue := range review.Issues {
        switch issue.Severity {
        case "critical":
            criticalCount++
        case "error":
            errorCount++
        case "warning":
            warningCount++
        }
    }
    
    // 有 critical 问题 → 整章重写
    if criticalCount > 0 {
        return "rewrite"
    }
    
    // error 问题占多数 → 整章重写
    if errorCount > warningCount {
        return "rewrite"
    }
    
    // warning 问题占多数 → 小范围修改
    if warningCount > errorCount {
        return "edit"
    }
    
    // 默认：根据 verdict 决定
    if review.Verdict == "rewrite" {
        return "rewrite"
    }
    return "edit"
}

// Execute 中使用
strategy := selectRepairStrategy(&r)
r.RepairStrategy = strategy
```

#### 4.3 修改 assets/prompts/writer.md

在"重写与打磨"段增加策略说明：

```markdown
## 重写与打磨

根据 `rewrite_brief.repair_strategy` 选择修复方式：

- **edit**（小范围修改）：
  - 使用 `edit_chapter` 精确替换问题段落
  - 只修改有 issues 的部分，保留其他内容
  - 适用于：AI 味套句、标题问题、轻微节奏问题

- **rewrite**（整章重写）：
  - 使用 `draft_chapter(mode="write")` 整章覆盖
  - 适用于：逻辑硬伤、结构性问题、严重节奏问题

不要混淆策略：小问题用大手术是浪费 turn，大问题用小修是治标不治本。
```

## 五、P1: 审阅-修复循环限流

### 问题

可能无限循环（审阅→修复→再审阅→...）。

### 方案

设置最大循环次数，超过后强制 accept 或标记需要人工干预。

#### 5.1 修改 domain/runtime.go

```go
// Progress 新增字段
ReviewRepairCycles map[int]int `json:"review_repair_cycles,omitempty"` // 每章的审阅-修复循环次数
```

#### 5.2 修改 store/progress.go

```go
// IncrementReviewRepairCycle 增加章节的审阅-修复循环次数
func (s *ProgressStore) IncrementReviewRepairCycle(chapter int) (int, error) {
    var count int
    err := s.io.WithWriteLock(func() error {
        p, err := s.loadUnlocked()
        if err != nil {
            return err
        }
        if p == nil {
            return nil
        }
        if p.ReviewRepairCycles == nil {
            p.ReviewRepairCycles = make(map[int]int)
        }
        p.ReviewRepairCycles[chapter]++
        count = p.ReviewRepairCycles[chapter]
        return s.saveUnlocked(p)
    })
    return count, err
}

// GetReviewRepairCycle 获取章节的审阅-修复循环次数
func (s *ProgressStore) GetReviewRepairCycle(chapter int) int {
    p, err := s.Load()
    if err != nil || p == nil || p.ReviewRepairCycles == nil {
        return 0
    }
    return p.ReviewRepairCycles[chapter]
}
```

#### 5.3 修改 tools/save_review.go

```go
const maxReviewRepairCycles = 3

// Execute 中检查循环次数
cycle, _ := t.store.Progress.GetReviewRepairCycle(r.Chapter)
if cycle >= maxReviewRepairCycles {
    // 强制 accept，标记需要人工干预
    finalVerdict = "accept"
    escalationReason = fmt.Sprintf("已达最大审阅-修复循环次数 (%d)，强制 accept", maxReviewRepairCycles)
    // 记录警告
    r.Issues = append(r.Issues, domain.ReviewIssue{
        Type:        "system",
        Severity:    "warning",
        Description: fmt.Sprintf("该章节已达到最大审阅-修复循环次数 (%d)，需要人工干预", maxReviewRepairCycles),
        Evidence:    fmt.Sprintf("review_repair_cycles=%d", cycle),
        Suggestion:  "请人工审阅并决定是否需要进一步修改",
    })
} else {
    t.store.Progress.IncrementReviewRepairCycle(r.Chapter)
}
```
