## P0 — 当前版本缺失的"必须功能"

### 1. 角色管理页补全（CharactersPage CRUD）

**现状**：CharactersPage.tsx 仅有角色展示功能（从 outline 读取），没有角色增删改查、角色卡片编辑、角色关系图编辑。

**实现方案**：

```
src/pages/CharactersPage.tsx    — Tab 容器（~120 行）
src/components/characters/
├── CharacterCard.tsx            — 角色卡片（名称、别名、角色定位、特质标签）
├── CharacterEditor.tsx          — 角色编辑模态框（表单 + 保存）
├── CastList.tsx                 — 配角名册列表 + CRUD
└── RelationGraph.tsx            — 关系图谱（力导向图，已有实现可复用）
```

**IPC 通道**（已就绪）：
- `getBookCharacters(id)` → `saveBookCharacters(id, chars)`
- `getBookCast(id)` → `saveBookCast(id, entries)`

**验收标准**：
- [ ] 角色卡片列表展示（名称、角色定位、核心特质、人物弧光）
- [ ] 新建角色表单（填写名称、别名、角色定位、描述、特质等）
- [ ] 编辑/删除角色
- [ ] 角色关系图谱可交互

### 2. 章节编辑器 Markdown 预览

**现状**：`ChapterPage.tsx` 使用纯 `<textarea>`，写作者看不到 Markdown 渲染效果。

**实现方案**：

- 安装 `react-markdown` 或 `marked`（轻量，无额外 runtime）
- 在编辑器下方增加 "预览" tab 或分屏模式
- 预览区渲染：标题、加粗、列表、代码块、引用

**验收标准**：
- [ ] 编辑器底部有"编辑/预览"切换 tabs
- [ ] 预览区正确渲染 Markdown 格式
- [ ] 预览区保持与编辑区同步滚动

---

### 3. 数据统计仪表盘

**现状**：系统积累了 `usage.json`（Token 消耗、成本）、`progress.json`（字数、进度）、`run.json`（运行元数据）等量化数据，但没有任何可视化面板。

**实现方案**：

新建 `src/pages/DashboardPage.tsx`，展示：

```
┌─────────────────────────────────────┐
│ 📊 写作统计                         │
├──────────────────┬──────────────────┤
│  总字数: 85,432  │  完成章节: 12/24 │
│  总 Token: 2.1M  │  总成本: $12.45  │
├──────────────────┴──────────────────┤
│  每日字数趋势（折线图/SVG）          │
│  ▁▃▂▅▇▆▄▁▃▂▄▆                      │
├─────────────────────────────────────┤
│  各章节字数分布（柱状图/SVG）        │
│  ████▇████▆▅▆███▇                   │
└─────────────────────────────────────┘
```

**数据源**：
- `progress.json` → `totalWordCount`, `completedChapters`
- `usage.json` → `totalInputTokens`, `totalOutputTokens`, `totalCostUSD`
- `chapters/` → 逐个统计字数

**验收标准**：
- [ ] 仪表盘页面在书籍导航中可访问
- [ ] 展示总字数、完成进度、Token 消耗、成本
- [ ] 每日字数趋势可视化
- [ ] 导航侧栏新增 "📊 统计" 条目

### 4. 数据备份与恢复

**现状**：所有数据存储在 `~/.ainovel-gui/`，没有一键备份/恢复功能。

**实现方案**：

新增 IPC 通道（`electron/ipc/system.ts`）：

```
backup-data → 生成 ~/.ainovel-gui/ainovel-backup-{date}.zip
restore-data(path) → 解压 zip 覆盖 ~/.ainovel-gui/
```

前端页面（设置在 `SettingsPage.tsx` 新增备份区域）：

```
┌───────────────────────────────┐
│ 💾 数据备份与恢复              │
│                               │
│ [导出数据快照]  ← 生成 ZIP    │
│  上次备份: 2026-07-01 14:30   │
│                               │
│ [从快照恢复]  ← 选择 ZIP 文件 │
│  ⚠ 将覆盖当前所有数据         │
└───────────────────────────────┘
```

**验收标准**：
- [ ] 导出备份生成 `.zip`，包含完整的 `ainovel.db` + `books/` 目录
- [ ] 从备份 ZIP 恢复后数据完整
- [ ] 设置页有备份/恢复的 UI 入口

## P1 — 重要效率提升

### 5. 全局搜索 Cmd+K

**现状**：用户无法跨模块搜索章节内容、角色名、事件等。

**实现方案**：

- 安装 `react-hot-keys` 或直接用 `useEffect` 监听 `Cmd+K`
- 新建 `src/components/CommandPalette.tsx` 模态搜索面板
- 搜索范围：章节标题+内容、角色名、时间线事件、大纲条目
- IPC 通道：新增 `search-all(bookId, query)` 在主进程跨表搜索

**交互流程**：

```
Cmd+K → 弹出搜索面板 → 输入关键词 → 结果分组展示
         ├── 📖 章节 (3)
         ├── 👤 角色 (2)
         ├── ⏳ 事件 (5)
         └── 📋 大纲 (1)
       → 点击结果跳转到对应页面
```

**验收标准**：
- [ ] `Cmd+K` 唤出搜索模态框
- [ ] 搜索结果覆盖章节/角色/事件/大纲
- [ ] 点击搜索结果正确导航

---

### 6. 写作目标与进度追踪

**现状**：没有目标设定，用户不知道今天写了多少字。

**实现方案**：

- 设置页新增"写作目标"区域（每日/每周目标字数）
- 工作台侧栏显示今日进度条
- 数据存储在 `config.json`（目标）+ 实时统计（当前字数）

**验收标准**：
- [ ] 设置页可配置每日/每周字数目标
- [ ] 工作台状态栏显示今日实际字数 vs 目标进度条
- [ ] 跨日自动重置

---

### 7. 章节版本对比 Diff

**现状**：章节在 draft → review → rewrite → final 的迭代过程中，用户无法对比两个版本的差异。

**实现方案**：

- 在 `ChapterPage.tsx` 新增"版本对比"模式
- 使用 `diff` 库（如 `diff` 或 `jsdiff`）做行级对比
- 显示：绿色=新增，红色=删除，黄色=修改

**验收标准**：
- [ ] 章节页有"对比"按钮，可选择对比版本（草稿 vs 终稿 / 当前 vs 上次）
- [ ] 差异以行级 add/remove/change 高亮显示
- [ ] 不影响正常编辑功能

## P2 — 体验打磨

### 8. 大纲页面可编辑

**现状**：OutlinePage.tsx 纯展示，无法增删改卷/弧/章的名称和核心事件。

**实现方案**：

- 每个大纲条目点击后进入编辑模式（inline edit 或 modal）
- 保存通过 `saveBookOutline(id, data)` IPC
- 支持：重命名卷/弧/章、修改核心事件、调整顺序

**验收标准**：
- [ ] 卷名、弧名、章名可双击编辑
- [ ] 保存后数据持久化
- [ ] 再次进入页面显示已保存的内容

---

### 9. 拖拽排序

**现状**：大纲卷→弧→章、章节列表、时间线事件均不支持拖拽排序。

**实现方案**：

- 安装 `@dnd-kit/core` + `@dnd-kit/sortable`
- 优先应用于：大纲页面（卷排序、弧排序、章排序）、章节列表
- 拖拽后自动保存新顺序

**验收标准**：
- [ ] 大纲的卷可拖拽排序
- [ ] 弧可拖拽到不同卷
- [ ] 章可拖拽到不同弧
- [ ] 排序后数据持久化

---

### 10. 评审雷达图可视化

**现状**：ReviewsPage.tsx 以网格卡片展示七维评分，不够直观。

**实现方案**：

- 用 SVG 绘制雷达图（七边形）
- 从 `review.dimensions[]` 读取各维度评分
- 颜色映射：consistency=绿, character=蓝, pacing=橙, ...

**验收标准**：
- [ ] 评审详情顶部显示七维雷达图
- [ ] 鼠标悬停显示具体分数
- [ ] 多个评审可叠加对比
